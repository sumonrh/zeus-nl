Summary of Material Model Implementation in ZEUS-NL

 

This document is a summary of material model implementation in ZEUS-NL. Source files are also provided for an example material model.

Example material model

Example material models are the bilinear elasto-plastic model with kinematic strain-hardening for steel material (stl1) and trilinear concrete model (con1). Following material names and numbers are used for example material models in ZEUS-NL.

¡¤         Material name : mm11 & mm12

¡¤         Material number : 11 & 12

Provided source code

The sample source files provided contain all necessary subroutines for the implementation of a new material model. Only the subroutines in these files are needed to be modified to meet the requirements of the new material model.

¡¤         Source file for the Reader of ZEUS-NL : mm11.f (for steel) & mm12.f (for concrete)

¡¤         Source file for the Solver of ZEUS-NL : MM11.f (for steel) & MM12.f (for concrete)

Implementation Procedure

1.      Modify/Update provided source FORTRAN files (mm11.f and MM11.f or mm12.f and MM12.f) based on your own material model

–        In both source files, carefully read the comments in each subroutine and understand how each subroutine works before getting started with the new model.

2.      After finishing coding, compile modified source files (refer to documentation on ¡°ZEUS-NL compilation procedure¡±)

Debug until the material model is validated.

 

Material Model Implementation in Zeus-NL

1-       Introduction

This chapter in the manual takes the user through the process of understanding the FORTRAN 77 subroutines that represent material constitutive models. The aim of this chapter is to make the user able to implement their own material models and/or modify existing ones. A brief overview of the implementation process will be discussed in the first section followed by a detailed description of all reader and solver, which are the two components of the ZEUS NL analysis engine, subroutines in the next two sections. Finally, an implementation example of a simple bilinear elasto-plastic steel model will be provided to highlight all necessary instructions and procedures in the implementation process.

2-       Overview

In order to implement a new material model, the user has to write two FORTRAN files: 1) a file for the Reader; and 2) a file for Solver. Each file contains a set of subroutines that need to be coded properly by the user to output the sought material behavior. These subroutines will be discussed shortly in the next two sections.

The Reader reads the the material data assigned in Zeus-NL input files with ¡°dat¡± extensions and generates a set of files to be read by the Solver, therefore the subroutines for the Reader have to:

(1)   Define the number of external variables (EXTVARs) and the number of internal variables (INTVARs) the user needs for the implemented material model. The following paragraph defines the meaning of the terms EXTVARs and INTVARs.

The EXTVARs are the variables assigned by the user in Zeus-NL input file (with ¡°dat¡± extension); these variables represent the material properties required to define the uni-axial material stress-strain relationship such as yield strength, elastic modulus, etc. The INTVARs are variables calculated inside the solver subroutines during the analysis at each time step. They are required to model the material cyclic behavior at each point on the stress-strain curve, examples of INTVARs are: current strain and stress values, stress and strain values at the previous loading step, current strain increment (which defines whether the increment is tensile or compressive, loading or unloading from the previous state), etc. The simple example of material model implementation at the end of this section will give more insight to the user on EXTVARs and INTVARs definitions.

(2)   Establish Young¡¯s Modulus ¡®E¡¯ of the material.

(3)   Check the validity of EXTVARs values, which are assigned by the user, from obvious errors and also be able to write error statements identifying the type of errors in an ¡°out¡± extension file created by the Reader.

The material model subroutines for the Solver calculates the stress values for the given strains and strain increments (loading and unloading strain cases). It also provides an appropriate value of the tangent stiffness of the material. Therefore, the Solver subroutines should be able to: (1) Set the initial values of the INTVARs; (2) Update the values of INTVARs for each load step; (3) Add the contribution of section forces; (4) Calculate the stress values; (5) Add the contribution of section stiffnesses; and (6) calculate the tangent modulus at any point on the stress-strain curve; for all monitoring points belonging to the material model of elements TYPE (1) and (2). TYPE (1) is the cubic elasto-plastic beam-column elements while TYPE (2) are cubic elasto-plastic retrofit elements.

The next two sections will discuss in more detail the material subroutines for the Reader and the Solver.

Subroutine ¡°scfrc3¡±

This subroutine adds the contribution to the section forces (for all monitoring points of elements TYPE(1)) by integrating stresses over the area of monitoring points. In this subroutine the needed material properties (EXTVARs) are called from the MAT(*) array. The pointer PSTRSS is used to locate the INTVARs in the array STRESS. Notice that inappropriate usage of the ¡°COMMON¡± command for setting the ¡°COMMON BLOCKS¡± can cause so many errors in your code, therefore is suggested that the user should set all shared ¡°COMMON BLOCKS¡± in all subroutines to have the same order of variables.

     SUBROUTINE SCFRC31(

     +                    MAT,

     +                    SEC,

     +                    STRESS)                          

 

C-----------------------------------------------------------------------

C     This subroutine adds the contribution to the section forces of

C     monitoring points belonging to material MODEL(1) in 3D cubic

C     elements.

C-----------------------------------------------------------------------

C     [stl1] Bilinear Elasto-Plastic Model

C     called by CUBFRC3 subroutine in part2.f

C     call STMDL1 subroutine in MM1.f

C-----------------------------------------------------------------------

 

      IMPLICIT NONE

 

C..........................[ ARRAYS AND POINTERS ]......................

C

C

C..........................[ SUBROUTINE VARIABLES ].....................

C

C     MGAUSY,MGAUSZ,FGAUS : section forces (generalized stresses (sf))

C     E      :YOUNG'S MODULUS (E)

C     YIELD2 : 2*(YIELD STRENGTH

C     MUE    : HARDENINIG MODULS (MU*E)

C     STRNO  : OLD STRAIN of each MP (monitoring point)

C     STRSO  : OLD STRESS of each MP

C     TENSO  : OLD TENSILE YIELD strength of each MP 

C     COMPO  : OLD COMPRESSIVE YIELD strength of each MP

C     STRN   : NEW STRAIN of each MP

C     STRS   : NEW STRESS of each MP

C     TENS   : NEW TENSILE YIELD strength of each MP

C     IE     : index number of assembling order

C

      REAL*8 MAT(*),DY,DZ,A,ADY,ADZ,SEC(*),STRESS(*),CURVY(2),CURVZ(2),

     +       EPS,MGAUSY(2),MGAUSZ(2),FGAUS(2),E,YIELD2,MUE,

     +       STRNO,STRSO,TENSO,COMPO,STRN,STRS,TENS

 

      INTEGER OUTPRC,OUTMON                                

 

      INTEGER PSTRSS

 

      INTEGER PNTMAT,J,NUMNT,PSEC,GAUSS,I1,TYP

C

C..........................[ VARIABLES NOT NEEDED ].....................

C

      INTEGER PSCMNT,IMAT

 

      REAL*8 A1,A2,A3,A4,A5,A6,GJ

      REAL*8 STRESS0(1000,5,2,2),STRAIN0(1000,5,2,2)

     

      REAL*8 STRESS1(1000,5,500,2),STRAIN1(1000,5,500,2)         

      REAL*8 SECP1(1000,5,500,2,5)                         

C

C..........................[     COMMON BLOCKS    ].....................

C

      COMMON /E31_5/ PSEC,NUMNT,PSTRSS,PNTMAT,PSCMNT,IMAT

      COMMON /E31_6/ J,GAUSS     

      COMMON /M_S1/ E,YIELD2,MUE,STRNO,STRSO,TENSO,COMPO,STRN,STRS,TENS

      COMMON /E31_3/ DY,DZ,A,ADY,ADZ

      COMMON /E31_1/ CURVY,CURVZ,EPS,MGAUSY,MGAUSZ,FGAUS,A1,A2,A3,A4,A5,

     +               A6,GJ

      COMMON /AS23_4/ I1,TYP

      COMMON /STRESS0/ STRESS0                                   

      COMMON /STRAIN0/ STRAIN0                                   

     

      COMMON /OUTPUT1/ OUTPRC,OUTMON                             

      COMMON /STRESS1/ STRESS1                                   

      COMMON /STRAIN1/ STRAIN1                                   

      COMMON /SECP1/ SECP1                                

C

C.......................................................................

 

CNOTE...................................................................

C     1) "NUMNT"  = number of monitoring points

C

C     2) "PNTMAT" = pointer in vector [MAT]

C

C     3) "PSTRSS" = pointer in vector [STRESS].

C

C         Note that "PSTRSS" must be continuously updated as the

C         individual variables of vector [STRESS] are picked up.

C         The value of "PSTRSS" must point at the last storage

C         position of a monitoring point once its contribution has

C         been calculated

C

C     4) "PSEC"   = pointer in vector [SEC]

C

C     5) Vector [SEC] contains (GJ) and

C        [ dy , dz , A , A*dy , A*dz , A*dy*dy , A*dy*dz , A*dz*dz ]

C        for each monitoring point, where:

C

C            - dy   = distance from centroid along y-axis

C            - dz   = distance from centroid along z-axis

C            - A   = stress area

C

C     6) Vector [MAT] contains the material properties for each of

C        the materials used

C

C     7) Vector [STRESS] contains ALL the storage required for

C        monitoring points including stresses, strains, and other

C        variables that may be required depending on the material model

CNOTE...................................................................

 

 

C     Establish the material properties [PRP] needed

      E=MAT(PNTMAT)           ! YOUNG'S MODULUS (E)

      YIELD2=2.*MAT(PNTMAT+1) ! 2*(YIELD STRENGTH)

      MUE=MAT(PNTMAT+2)*E     ! HARDENINIG MODULS (MU*E)

             

 

 

C     Loop over all the monitoring points at the two Gaussian sections

C

      DO J=1,NUMNT

 

C       Establish the distances to the centroid "DY" and "DZ", the

C       stress area "A", and "ADY=A*DY" and "ADZ=A*DZ" for monitoring

C       point(J)

C

        DY=SEC(PSEC+1)

        DZ=SEC(PSEC+2)

 

        A=SEC(PSEC+3)

 

        ADY=SEC(PSEC+4)

        ADZ=SEC(PSEC+5)

 

        PSEC=PSEC+8

 

 

        DO GAUSS=1,2

 

CNOTE...................................................................

C     Material Model 1 (stl1) : monitoring pts = 6 <- modprp(2,1) = 6

C      STRESS(1) : OLD STRAIN        (STRNO)

C      STRESS(2) : OLD STRESS        (STRSO)

C      STRESS(3) : OLD TENSILE YIELD (TENSO)

C      STRESS(4) : NEW STRAIN        (STRN)

C      STRESS(5) : NEW STRESS        (STRS)

C      STRESS(6) : NEW TENSILE YIELD (TENS)

CNOTE...................................................................

 

C         Establish the variables required for monitoring point(J) 

C         to calculate the stress "STRS" and the current tensile 

C         yield "TENS"

C

          PSTRSS=PSTRSS+1

          STRNO=STRESS(PSTRSS)            ! OLD STRAIN

 

          PSTRSS=PSTRSS+1

          STRSO=STRESS(PSTRSS)            ! OLD STRESS

 

          PSTRSS=PSTRSS+1

          TENSO=STRESS(PSTRSS)            ! OLD TENSILE YIELD

 

          COMPO=TENSO-YIELD2        ! OLD COMPRESSIVE YIELD

 

 

C         Find the new strain "STRN"

          PSTRSS=PSTRSS+1

          STRESS(PSTRSS)=DY*CURVY(GAUSS)+DZ*CURVZ(GAUSS)+EPS

          STRN=STRESS(PSTRSS)       ! NEW STRAIN

 

 

C         Find the new stress "STRS" & update the tensile yield "TENS"

C

          CALL STMDL1

 

          MGAUSY(GAUSS)=MGAUSY(GAUSS)+ADY*STRS

          MGAUSZ(GAUSS)=MGAUSZ(GAUSS)+ADZ*STRS

          FGAUS(GAUSS)=FGAUS(GAUSS)+A*STRS

 

          PSTRSS=PSTRSS+1

          STRESS(PSTRSS)=STRS       ! NEW STRESS

 

          PSTRSS=PSTRSS+1

          STRESS(PSTRSS)=TENS       ! NEW TENSILE YIELD

 

CNOTE...................................................................

C     1) "PSTRSS" has increased by "6" which is the specified amount

C        of storage for material "MODEL1"

C     2) For other material models, the important variable to determine

C        is the new stress "STRS". However, it might be necessary to

C        update other variables relevant to the material model

CNOTE...................................................................

 

 

          IF(J.EQ.1) THEN

             STRESS0(I1,IMAT,1,GAUSS) = STRS

             STRAIN0(I1,IMAT,1,GAUSS) = STRN

          ENDIF

 

          IF(J.EQ.NUMNT) THEN

             STRESS0(I1,IMAT,2,GAUSS) = STRS

             STRAIN0(I1,IMAT,2,GAUSS) = STRN

          ENDIF

 

          IF((OUTMON.EQ.1).OR.OUTMON.EQ.2) THEN                  

            STRESS1(I1,IMAT,J,GAUSS) = STRS

            STRAIN1(I1,IMAT,J,GAUSS) = STRN

             

            SECP1(I1,IMAT,J,GAUSS,1)=DY

            SECP1(I1,IMAT,J,GAUSS,2)=DZ

            SECP1(I1,IMAT,J,GAUSS,3)=A

            SECP1(I1,IMAT,J,GAUSS,4)=ADY

            SECP1(I1,IMAT,J,GAUSS,5)=ADZ

          ENDIF            

 

        END DO

 

      END DO

 

      RETURN

      END

 SUBROUTINE ¡°STMDL¡±

This subroutine calculates the stress values at all monitoring points of elements TYPE(1) for any given strain value, increment and internal variables (INTVARs).

SUBROUTINE STMDL1                                    

 

C-----------------------------------------------------------------------

C     This subroutine calculates the stress at a monitoring point for

C     material MODEL(1).

C-----------------------------------------------------------------------

C     [stl1] Bilinear Elasto-Plastic Model

C     called by SCFRC31 subroutine in MM1.f

C-----------------------------------------------------------------------

 

      IMPLICIT NONE

 

C..........................[ ARRAYS AND POINTERS ]......................

C

C

C..........................[ SUBROUTINE VARIABLES ].....................

C

      REAL*8 E,YIELD2,MUE,STRNO,STRSO,TENSO,COMPO,STRN,STRS,TENS,

     +       DSTRN,DSTRN1,DELS

C

C..........................[ VARIABLES NOT NEEDED ].....................

C

C

C..........................[     COMMON BLOCKS    ].....................

C

      COMMON /M_S1/ E,YIELD2,MUE,STRNO,STRSO,TENSO,COMPO,STRN,STRS,TENS

C.......................................................................

 

 

CNOTE...................................................................

C     Material Model 1 (stl1) : monitoring pts = 6 <- modprp(2,1) = 6

C      STRESS(1) : OLD STRAIN        (STRNO)

C      STRESS(2) : OLD STRESS        (STRSO)

C      STRESS(3) : OLD TENSILE YIELD (TENSO)

C      STRESS(4) : NEW STRAIN        (STRN)

C      STRESS(5) : NEW STRESS        (STRS)

C      STRESS(6) : NEW TENSILE YIELD (TENS)

CNOTE...................................................................

 

 

C     Determine the portion of the stress-strain curve on which the

C     stress state will vary according to the original stress state and

C     the strain increment

 

      DSTRN=STRN-STRNO  !STRAIN INCREMENT

 

 

 

      IF(DSTRN.GE.0.0) THEN   ! TENSILE STRAIN INCREMENT

 

        IF(STRSO.EQ.TENSO) THEN           ! ORIGINAL TENSILE YIELD STATE

 

          DELS=MUE*DSTRN

          STRS=STRSO+DELS

          TENS=STRS    

 

 

        ELSE

 

C         Determine the strain increment "DSTRN1" needed to bring the

C         stress state to tensile yield

C

          DSTRN1=(TENSO-STRSO)/E

       

 

          IF(DSTRN.GE.DSTRN1) THEN  ! TENSILE YIELD IS REACHED

 

            DELS=MUE*(DSTRN-DSTRN1)

            STRS=TENSO+DELS

            TENS=STRS

 

 

          ELSE                      ! TENSILE YIELD IS NOT REACHED

 

            STRS=STRSO+E*DSTRN

            TENS=TENSO

 

          END IF

 

        END IF

 

      ELSE              ! COMPRESSIVE STRAIN INCREMENT

 

        IF(STRSO.EQ.COMPO) THEN           ! ORIGINAL COMPRESSIVE YIELD STATE

 

          DELS=MUE*DSTRN

          STRS=STRSO+DELS

          TENS=STRS+YIELD2   

 

        ELSE

 

C         Determine the strain increment "DSTRN1" needed to bring the

C         stress state to compressive yield

C

          DSTRN1=(COMPO-STRSO)/E

 

          IF(DSTRN.LE.DSTRN1) THEN  ! COMPRESSIVE YIELD IS REACHED

 

            DELS=MUE*(DSTRN-DSTRN1)

            STRS=COMPO+DELS

            TENS=STRS+YIELD2

 

 

          ELSE                      ! COMPRESSIVE YIELD IS NOT REACHED

 

            STRS=STRSO+E*DSTRN

            TENS=TENSO

 

 

          END IF

 

        END IF

 

      END IF

 

 

      RETURN

      END

This subroutine calculates the tangent stiffness at any point on the stress-strain curve.

      SUBROUTINE ETMDL1                                    

 

C-----------------------------------------------------------------------

C     This subroutine calculates the tangent modulus of the

C     stress-strain curve for material MODEL(1).

C-----------------------------------------------------------------------

C     called by SCSTF31 subroutine in MM1.f

C-----------------------------------------------------------------------

 

      IMPLICIT NONE

 

C..........................[ ARRAYS AND POINTERS ]......................

C

C

C

C..........................[ SUBROUTINE VARIABLES ].....................

C

      REAL*8 STRS,TENS,COMP,E,MUE,ET

C

C..........................[ VARIABLES NOT NEEDED ].....................

C

      REAL*8 YIELD2

C

C..........................[     COMMON BLOCKS    ].....................

C

      COMMON /M_E1/ E,YIELD2,MUE,STRS,TENS,COMP,ET

C

C.......................................................................

C     compression, use the elastic modulus; else, use the strain

      IF(SNGL(STRS).LT.SNGL(TENS).AND.SNGL(STRS).GT.SNGL(COMP)) THEN

      END

 

5-       Simple Example

The implementation process of stl1 (bi-linear elasto-plastic with kinematic hardening) material model is briefly discussed. This section will mainly focus on the coding part of the material model in subroutines STMDL and ETMDL, which calculate the stress and tangent stiffness, for any loading/unloading case, respectively. Other subroutines that require modification were discussed in detail in Sections 3 and 4.

stl1 model contains 3 EXTVARs that are required to define the material behavior and are assigned by the user in the input file. These parameters are: (1) E, Young¡¯s Modulus; (2) YIELD, Yield Strength (sy); and (3) MUE, Strain-hardening parameter (m).

The model contains 6 INTVARs which are:

(1)   Old strain (STRNO)

(2)   Old stress (STRSO)

(3)   Old tensile yield stress (TENSO)

(4)   New strain (STRN)

(5)   New stress (STRS)

(6)   New tensile yield stress (TENS)
